# 3. 持久化

RDB快照：fork子进程，通过保存某一时间点内存快照实现，使用BGSAVE命令进行RDB的生成

AOF方式：fork子进程，将每一个写操作进行记录，并在服务器重启后进行重放。

一般推荐使用AOF模式，并将其fsync策略设为每秒。同时定期做RDB的备份
